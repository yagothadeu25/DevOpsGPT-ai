passo a passo
1
Iniciar o servidor K8sGPT
Sobe o servidor HTTP que vai receber as requisições do frontend. Precisa estar autenticado com um LLM backend previamente.
k8sgpt auth add --backend ollama --model llama3
k8sgpt serve --port 8080
2
Frontend chama POST /v1/analyze
O dashboard envia uma requisição com o namespace alvo e a flag explain: true para receber a análise detalhada do LLM.
POST http://localhost:8080/v1/analyze
3
K8sGPT escaneia o cluster via kubectl
Os analyzers internos inspecionam Pods, Services, HPAs, PVCs e Deployments. Dados sensíveis são anonimizados antes de seguir para o LLM.
4
LLM gera diagnóstico e solução
O contexto anonimizado é enviado ao LLM configurado (Ollama, OpenAI, etc.). O modelo retorna a causa raiz e os passos de correção em linguagem natural.
5
Dashboard renderiza os resultados
O frontend exibe os issues agrupados por severidade (critical, error, warning, info), com filtros e busca em tempo real. Cada card expansível mostra o AI analysis completo.
